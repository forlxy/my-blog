Deployment on AWS

For the experience of first use of AWS:

Initialisation

  1. Register a new AWS account.
  2. Open the AWS calculator to make sure the services chosen are in free-tier.
  3. Start from creating application in the Elastic Beanstalk service.
  4. Continue to create a new environment.
  5. While waiting for its creating process, navigate to the RDS panel to create a new database, which is potentially a SQL Server database. Then, deploy it.
  6. Write down the endpoint, port, username and password.
  7. Open the SSMD on your computer, try to connect to the server with following settings:
    1. Engine
    2. endpoint,port
    3. username
    4. password
  8. Create a new database and populate your data into it.
  9. Write down the database name, and assemble the connection string in following format: Data Source=,;Initial Catalog=;User ID=;Password=;

Configuration

  1. Navigate to the IAM panel. Create a new user and a new group, then download the credential file(.csv).
  2. Waiting for them to finish….
  3. Open the environment which just finished creation, select the Configure panel on the left, select the Modify button of Network to generate an public IP address for it.
  4. Now, go to the VPC panel, select the Security Groups under the Security label.
  5. Select the default one and then choose the Actions -> Editing Inbound rules.
  6. Change to All TCP, and add your IP in the Source.

Host your website

  1. Now, install the AWS toolkit for .Net.
  2. Open the VS, load your workspace. Right click your project and then select the publish to the AWS.
  3. Click the profile button and then import from the csv file which was downloaded at step 6.
  4. Be aware of the region you choose, it do really matters.
  5. Select the environment and then deploy it.
  6. Now go to your environment, wait for its Health turning to green, and then you can open the URL to check if everything is perfect.
  7. Go to the EC2 Dashboard and choose your instance, write down the IPV4 Public address.
  8. Register a new domain….
  9. Add the “www A” and “ A” to its DNS records.
  10. Now your can access your website by visiting your domain.
